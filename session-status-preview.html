Session Status Report — Table Style Preview

Five different visual directions for the sections × sessions status grid, all rendered from the real, live SCHEDULE data in js/data.js — same data as session-status-report.html, no snapshots. Each session's date/time is shown where the cell has room, and always available on hover (title tooltip) everywhere else. Scroll through and tell me which one (or which pieces of which) to carry into the real page.

01 · Print Spreadsheet

Plain report grid

Black hairline borders, white background, blue row labels, green checkmarks — closest to the reference image this report started from. Each cell now shows the session's date on a second line; hover any cell for the full day + time.

02 · Ops Dashboard (current)

What session-status-report.html uses today

Dark/monospace, matching class-program.html / teacher-schedule.html / session-breakdown.html's shared theme — grade-colored left border per row, cyan headers, green checkmarks, small muted date under each mark (same "day-name/date-num" treatment already used elsewhere on the site). Hover for the full day + time.

03 · Progress Dots

Circles instead of checkmarks

No table borders — each row is a light card with 5 small circles (filled = done, hollow = upcoming), reusing the same "session-num-badge" circle treatment already used on session-breakdown.html. Too small to print date/time inline, so it's on hover only — the tradeoff for the friendlier, less spreadsheet-y look.

04 · Heatmap Blocks

Solid-color scan view

Each cell is a solid colored block (green = done, slate = upcoming) with the session number and a tiny date on two lines, no glyphs to squint at. Densest and fastest to scan at a glance across many sections — hover for the full day + time.

05 · Progress Bar Summary

Completion at a glance

One segmented bar per section (5 slices, filled left-to-right) plus an "X/5" fraction — trades exact per-session identity for the fastest possible "how close is this section to finishing" read. Each segment is individually hoverable for that session's date/time; best as a companion/summary view, not a full replacement for the detailed grid.
