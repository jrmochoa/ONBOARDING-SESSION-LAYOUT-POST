This is a preview page, not the LMS. Below are three ready-to-paste snippets — pick the one that matches the Moodle site you're editing. Each one is an <iframe> plus a small <script> that auto-resizes the frame to exactly match the embedded page's real height, so there's no internal scrollbar and no dead empty space — the widget just grows/shrinks like normal page content. This does require the <script> tag to survive Moodle's editor (paste via Edit HTML source, the "<>" button, not the rich-text box) — the rest of the styling still needs no <style> tag at all, since that lives inside class-program.html itself. If your Moodle strips even source-mode scripts, it just falls back to the fixed-height, internally-scrolling box it was before — not broken, just less polished.

Live preview (both grades, all sections) — auto-height, no scroll

For sbcjhs1.aralinks.net (Grade 7 & 8) — pre-selects the Grade 7 tab on load (viewers can still click over to Grade 9 manually; this just controls what shows first): 
<iframe
  id="ace-class-program-frame"
  title="ACE Class Program Schedule — Grade 7"
  loading="lazy"
  style="width:100%; height:600px; border:1px solid #d0d0d0; border-radius:8px; display:block;"
></iframe>
<script>
(function(){
  var frame = document.getElementById('ace-class-program-frame');
  frame.src = 'https://jrmochoa.github.io/ONBOARDING-SESSION-LAYOUT-POST/class-program.html?grade=G7&_cb=' + Date.now();
  window.addEventListener('message', function(e){
    if(e.origin !== 'https://jrmochoa.github.io') return;
    if(!e.data || e.data.source !== 'ace-schedule-iframe') return;
    if(frame && typeof e.data.height === 'number') frame.style.height = e.data.height + 'px';
  });
})();
</script>

height:600px in the <iframe> tag is only the placeholder shown before the first height message arrives (roughly instant) — after that, the script keeps it sized to the real content, including when you switch grade/section tabs or toggle light/dark inside the frame. The id="ace-class-program-frame" must match between the iframe and the script if you ever rename it. Since this iframes the live GitHub Pages site directly (not a snapshot), it always reflects whatever's currently on class-program.html — no regenerating needed when the schedule changes.
